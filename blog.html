1. What are the differences between Local Storage & Session Storage

Answer:

  • Local Storage stores data in browser without any expiration date. Session storage is temporary.
  • Data is not deleted in Local Storage when browser is closed. On the other hand the data of Session storage expires when browser is close.
  • The capacity of Local Storage is 10MB & Session Storage is 5MB.

    • 2. What are the differences between Global Scope & Block Scope

    Answer:

  • Global scope is the default scope for all code running in script mode and Block scope id the scope created with a pair of curly braces
  • In a programming environment, the Global Scope is declared globally. It contains and is visible in all other scopes. The block scope of a variable means that the variable is accessible only within the block that is between the curly braces.
  • Variables in Global Scope can not access into Block Scope, but Variables in block scope can access all the data of Global scope if needed.
  • In both Global and Block scope we shouldn't use the 'var' variable because it does hoisting.

    • 4. By how many ways we can get 'undefined'

    Answer:
    8 reasons of returning undefined are described below:

    1. Variable that is not initialized will give 'undefined'
    2. Function with no 'return' will give 'undefined'
    3. Parameter that is not passed will be undefined
    4. If return has nothing on the right side will return undefined
    5. Trying to access into array elements outside of the index range will return undefined
    6. Property that doesn't exist in an object will give you undefined
    7. Deleting an element inside an array will return undefined
    8. Set a value directly to 'undefined' will return undefined
